# 图形选择决策规则

## 优先级系统

```
P1 用户明确指定图形类型       → 直接使用
P2 用户目标关键词             → 问题类型映射
P3 关系类型分析               → 关系类型匹配
P4 实体数量                   → 5+推荐系统图/树状图
P5 默认树状图                → 万能兜底
```

## 关键词 → 图形映射

| 用户意图 | 关键词 | 图形 | 优先级 |
|----------|--------|------|:------:|
| 分析原因 | 为什么、原因、失败、复盘、为什么上不去 | ③ 深挖图 | P2 |
| 梳理流程 | 如何、步骤、流程、怎么做、方法 | ⑤ 流程图 | P2 |
| 比较选择 | 区别、哪个好、对比、选择、vs、优劣 | ④ 比较图 | P2 |
| 分类体系 | 有哪些、分类、体系、构成、组成 | ② 树状图 | P2 |
| 规划战略 | 目标、规划、战略、方向、计划 | ⑦ 金字塔图 | P2 |
| 价值交换 | 商业模式、价值、赚钱、交换 | ① 交换图 | P2 |
| 融合定位 | 共同点、融合、交集、定位 | ⑥ 重叠图 | P2 |
| 看系统 | 系统、整体、蓝图、架构、全貌 | ⑧ 系统图 | P2 |
| 看时间线 | 历史、历程、时间线、演变、成长 | ⑨ 时间线图 | P2 |

## 问题类型 → 图形

| problem_type | 图形 |
|-------------|:----:|
| causal_analysis | ③ 深挖图 |
| process_mapping | ⑤ 流程图 |
| comparison | ④ 比较图 |
| classification | ② 树状图 |
| strategy_planning | ⑦ 金字塔图 |
| value_exchange | ① 交换图 |
| overlap_analysis | ⑥ 重叠图 |
| system_mapping | ⑧ 系统图 |
| timeline | ⑨ 时间线图 |

## 关系类型 → 图形

| 核心关系类型 | 图形 |
|------------|:----:|
| 因果 | ③ 深挖图 |
| 顺序/流程 | ⑤ 流程图 |
| 分类/层级 | ② 树状图 |
| 比较 | ④ 比较图 |
| 交换/交易 | ① 交换图 |
| 重叠/交集 | ⑥ 重叠图 |
| 层级/包含 | ⑦ 金字塔图 |
| 依赖/反馈 | ⑧ 系统图 |
| 时间顺序 | ⑨ 时间线图 |

## 实体数量 → 图形

| 实体数量 | 推荐 | 说明 |
|:-------:|:----:|------|
| 1~3 | 深挖图/重叠图 | 焦点问题或两两关系 |
| 4~6 | 交换图/比较图/时间线 | 中度复杂度 |
| 7~12 | 树状图/流程图/系统图/金字塔 | 高复杂度，需分组理解 |
| 13+ | 系统图（建议自动拆图） | 超过 20 节点自动拆多图 |

## 节点数量限制

| 图形 | 默认 | 最大 | 超限处理 |
|:----:|:---:|:---:|---------|
| 树状图 | 6~10 | 15 | 剪枝，保留主干 |
| 流程图 | 5~10 | 12 | 合并步骤 |
| 系统图 | 7~14 | 20 | 拆子系统图 |
| 时间线图 | 5~8 | 10 | 合并时段 |
| 比较图 | 4~7 | 8 | 减少维度 |
| 深挖图 | 5~9 | 12 | 合并表层原因 |
| 交换图 | 3~6 | 8 | 抽象同类主体 |
| 重叠图 | 3~5 | 6 | 去掉边缘概念 |
| 金字塔图 | 5~10 | 14 | 减少每层分支 |
