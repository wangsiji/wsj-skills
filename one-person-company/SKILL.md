# 一人公司三线统筹

> 用户（前百度算法→一人公司大理）三条收入/创造线的统筹框架。

## 核心方法论

### 优先级梯队（2026-07-21 确定）

```
P0：📝 秋秋很开心（恢复稳定更新，每周1-2篇）
P1：📖 网文创作（先动笔再谈好坏，每日15min/500字）
P2：✍️ 思吉账号 / 💻 独立开发 / 其他（内容稳定后启动）
```

### 「学什么就分享什么」

不追热点、不猜选题、不靠数据验证价值。用户正在学/做的东西就是内容素材，分享是学习的副产品。数据好是 bonus，数据差也不焦虑——学习本就没停。

**学习引擎管线**：
```
你学什么 → 深度研究 → 沉淀到 vault(skill+卡片) → 公众号发出（自媒体）
                                               ↘ 打包工具/模板（独立开发）
                                               ↘ 故事灵感（网文素材）
```

## 三条线

### 周节奏

| 时段 | 事项 | 备注 |
|------|------|------|
| 🏃 晨间 | 跑步 | 见 wsj-running-coach，P0不妥协 |
| 📝 写作日（周一~四） | 公众号 1 篇/周 + 碎片整理 | 学什么就写什么 |
| 📖 网文（每日 15min） | 500-1000 字 | 最小启动量，先完成再完美 |
| 🗓 周五~六 | 弹性（复盘/补更/探索） | 不设 KPI |

### 📝 自媒体（秋秋很开心）
- **粉丝**：~100 万（老婆账号移交，用户亲自执笔）
- **平台**：公众号（抖音/小红书未启）
- **历史验证**：✅ 2025-12 10+ 篇均阅读过万。断更半年后重启掉量——**是断更杀流量，非定位问题**
- **方向**：学习成长 + 好物 + 生活
- **方法**：「学什么就分享什么」——用户正在学的东西就是下一篇内容
- **节奏**：稳定 > 高频，先恢复每周 1-2 篇
- **首发话题候选**：知识卡片系统 / Obsidian + AI 知识管理 / 一人公司工具链
- **发布流程**：用户本人直接以秋秋开心身份写作，经老婆润色后发布（非代运营）

### ✍️ 自媒体（思吉）
- **状态**：**尚未开始写，无压力不设截止**
- **优先级**：随意——用户明确「思吉账号随意」

### 💻 独立开发
- **状态**：做过复利计算器小程序（试水），无持续收入
- **思路**：不是先想"做什么产品"，而是用户学/做的东西里哪个可以打包卖
- **候选**：COROS 数据看板、第二大脑 Obsidian 模板包、Hermes Shell 主题皮肤

### 📖 网文创作
- **状态**：**高一至今 10 年+读者，从未动笔**
- **类型**：系统文（用户偏好，求无脑爽感）
- **破局**：先动笔，写 1000 字就算赢，不追求第一篇就够格发布
- **关键**：不欠文采/灵感，欠「10 年读者高标准 vs 0 天作者第一稿」之间的落差认知。灵感不是等来的，是写出来的。
- **starter pack**：三部曲「🅰 都市系统（被裁算法工程师绑定了人生优化系统）」「🅱 穿越系统（社畜穿越异世界绑定内卷系统）」「🅲 现代游戏（大学生手机变现实游戏系统）」——选一个直接开写

## Never Do

- ❌ 追热点选题（用户要学什么写什么，不是市场热什么写什么）
- ❌ 编造用户经历/数据
- ❌ 建议用户做 AI 自媒体（用户对前沿技术兴趣不大，想做产品不是追逐热帖）
