## 风格选择

**默认风格 A（Kindle 纸书风）**。用户不提风格时直接用 A。

**风格 D（Apple 极简白风）** — 当用户说「苹果风」「Notion风」「Linear风」「飞书文档风」「极简风」「白色背景」「现代极简」「公众号」「小红书」「知识卡片」时使用。偏好：白色背景 + Apple系统字体 + 浅蓝#007AFF点缀色 + 细边框圆角卡片 + 模块化网格 + 大量留白。

**风格 C（深色信息长图风）** — 当用户说「信息图」「长图」「像这样的」并附参考图时使用。

## 布局结构

### 风格 C：深色信息长图风

深色背景 + 白色圆角卡片 + 分区灰底条目。适合信息图、教程长图、知识流程展示。

| 属性 | 值 |
|------|-----|
| 外背景 | `#0b0b12` |
| 卡片底色 | `#ffffff` |
| 卡片圆角 | 20px |
| 卡片阴影 | `0 16px 48px rgba(0,0,0,0.5)` |
| Header 背景 | `linear-gradient(160deg, #201535, #352552, #2d1d45)` |
| 条目底色 | `#f8f7fb` |
| 条目圆角 | 10px |
| 主标题色 | `#2a2a3a` |
| 正文字 | `#6a6a7a` |
| 小字色 | `#a0a0b0` |
| 分区标题圆点 | 6px 圆形 |

**布局结构**（从上到下）：
1. 深紫渐变 Header — 标签栏(TAG+NO.) / 主标题 / 副标题
2. 核心闭环 — 灰底条目卡片（圆点+标题+列表）
3. 分步流程 — 编号色块(蓝/绿/橙/紫) + 图标 + 要点
4. 使用场景 — 灰底条目(左侧图标+文字)
5. 核心洞察/金句 — 灰紫底居中斜体
6. 底部三栏网格 — 图标+标题+描述
7. Footer 分割线 + 标语

**配色分区**：
- 紫色 `#9b7fd4` — 理论/洞察
- 蓝色 `#6a8fc4` — 流程步骤
- 绿色 `#6abf8a` — 使用场景
- 橙色 `#d4946a` — 步骤3
- 粉色 `#b88ad4` — 步骤4

**典型卡片宽度**：480-500px（手机视图优先）。
```
┌─────────────────────────────────────┐
│  Header (h-128px, #3c3a37)         │
│  [TAG]  NO.001                      │
│  标题（font-black, text-3xl）       │
│  SUBTITLE                           │
├─────────────────────────────────────┤
│  Body (#f5f0eb)                      │
│  ▎ 摘要（加粗黑体，暖色竖条）       │
│  机制（灰底小字）                    │
│  ◈ 行动 / 对比分区                   │
│  "金句"                             │
│  Footer: BY 作者名                   │
└─────────────────────────────────────┘
```

### 风格 D：Apple 极简白风

白色背景，Apple/Notion/Linear 审美。适合公众号/小红书知识卡片、工具书风格信息图。

| 属性 | 值 |
|------|-----|
| 外背景 | `#f5f5f7` |
| 卡片底色 | `#ffffff` |
| 卡片圆角 | 32px |
| 卡片阴影 | `0 2px 40px rgba(0,0,0,0.06)` |
| 主色/强调色 | `#007AFF`（苹果蓝） |
| 辅色 1 | `#34C759`（苹果绿） |
| 辅色 2 | `#FF9500`（苹果橙） |
| 辅色 3 | `#AF52DE`（苹果紫） |
| 辅色 4 | `#FF3B30`（苹果红，错误/警示） |
| 主标题色 | `#1d1d1f` |
| 正文字 | `#515154` |
| 辅助文字 | `#6e6e73`/`#86868b` |
| 分割线/边框 | `#f0f0f0`/`#e8e8ed` |
| 灰底/浅色块 | `#f5f5f7` |
| 强调背景(蓝) | `linear-gradient(135deg, #e8f4fd, #f0f7ff)` |
| 字体 | `'Noto Sans SC', -apple-system, 'Inter'` |

**布局结构**（4:5 比例，720x1280px 或 800x1000px 基准）：

1. **Header** — 左侧大标题+英文副标题 / 右侧头像(或SVG线稿)+引用金句气泡，底部细分隔线
2. **核心金句** — 浅蓝渐变背景条 + 左侧蓝色竖条 + 一句话核心观点
3. **模块网格** — 每个模块有编号圆角badge(①-⑧)+标题+内容卡片
4. **2列并排** — 常见误区(浅红底)+使用建议(浅绿底)左右并排
5. **流程图** — 4个彩色渐变色块横向排列，箭头连接；或纵向流程箭头
6. **延伸知识** — 圆形 tag 胶囊(pill)样式排列
7. **底部标签** — 浅灰圆角方块标签(#标签名)

**配色分区**：
- 蓝色 `#007AFF` 模块①定义、④示例
- 绿色 `#34C759` 模块②价值、⑦建议
- 橙色 `#FF9500` 模块③流程
- 紫色 `#AF52DE` 模块⑤原理
- 红色 `#FF3B30` 模块⑥误区
- 青色 `#5AC8FA` 模块⑧延伸

**关键尺寸**：
- 卡片宽度 720-800px，高度 1000-1280px（4:5）
- Header padding: 28px 36px 14px
- 模块间距: 14px
- 正文字号: 12-13px
- 小字号: 9-10px
- 圆角: 大32px / 中12-16px / 小8-10px

### 风格 D 变体：学术知识卡配色

当用户说「知识卡片」「学习方法」「教程」「信息图」且**没有**明确要求「苹果风」「设计感」时，优先用此变体。当用户说「没有 GPT 做的好」说明当前配色太亮/太苹果，也应切换到学术配色。

| 属性 | 苹果蓝（原风格 D） | 学术知识卡（变体 D2） |
|------|-------------------|----------------------|
| 外背景 | `#f5f5f7` | `#F7F9FC` |
| 卡片圆角 | 32px | 10px |
| 卡片阴影 | 大而柔 | 小而聚焦 |
| 主标题色 | `#1d1d1f` | `#1A3B6C`（藏蓝） |
| 正文字 | `#515154` | `#4A4A4A` |
| 强调蓝 | `#007AFF` | `#4EA1EC` |
| 强调绿 | `#34C759` | `#21AC6B` |
| 强调橙 | `#FF9500` | `#F2A841` |
| 强调紫 | `#AF52DE` | `#C993CE` |
| 核心思想底 | 浅蓝渐变 | `#FEEBC2`（浅橙底） |
| 流程色 Step1~4 | 蓝/绿/橙/紫渐变 | `#6CC09D`/`#7BB1E8`/`#F2C078`/`#D7ADCC` |

```
┌─────────────────────────────┐
│  Header (padding 32px 36px) │
│  [TAG] ────── NO.001        │
│  标题（28px, bold, #3d3a37）│
│  SUBTITLE                    │
├─────────────────────────────┤
│  Body (padding 0 36px)       │
│  ┌─ 摘要（圆角灰底块） ──┐  │

│  ┌─ 机制（圆角灰底块） ──┐  │

```
