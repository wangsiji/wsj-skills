# 竞品分析驱动的小程序迭代工作流

## 何时触发
收到竞品分析报告或用户提出"改进产品"类需求时。

## 工作流

### Phase 1: 解析报告，提取可执行项
从报告中提取：
- **具体功能建议**（非抽象方向）—— 如"增加待办清单"而非"提高用户留存"
- **优先级标注**（P0/P1/P2）
- **实现复杂度评估**（纯前端 / 需后端 / 需第三方服务）
- **依赖关系**（A 依赖 B 先完成）

### Phase 2: 按可行性排序实施
```
纯前端可行 → 立即实施
需后端但可建 UI 预留 → 建 UI + 标注接口
需第三方服务 → 建 UI + 标注集成点
需真实数据 → 模拟数据 + 标注数据源
```

### Phase 3: 实施模式
每个功能项按 4 步交付：
1. 创建独立页面/组件
2. 注册路由（pages.json）
3. 添加入口链接（从现有页面链接过去）
4. Build 验证零报错

### Phase 4: 输出清单
格式化交付：功能名 + 状态 + 依赖说明

## 本项目的实践案例

### 输入：竞品分析报告（Flomo/小光芒/小日常/OffScreen/懒猫存钱等7款）
### 输出映射：

| 竞品 | 提取功能 | 优先级 | 实现 |
|------|---------|--------|------|
| Flomo | 待办清单 | P0 | timebox.vue 嵌入式 Todo |
| Flomo | 统计卡片 | P0 | mine.vue 4格统计 |
| 小光芒 | 储蓄计划 | P1 | savings.vue 新页面 |
| 小日常 | 白噪音 | P1 | timebox.vue 内嵌音频面板 |
| OffScreen | 成就系统 | P2 | achievements.vue 新页面 |
| OffScreen | 自习室 | P2 | studyroom.vue 新页面 |
| 懒猫存钱 | 会员订阅 | P1 | membership.vue 新页面 |
| 综合 | 微信登录 | P2 | profile.vue + uni.login |
| 综合 | 内容充实 | P0 | mine.vue 5+6篇 demo |

### 关键教训
- **不要等后端**：先建 UI + 模拟数据，标注接口预留点，后端就绪后只换数据源
- **闭环优先**：新增页面必须能从现有页面到达，导航链路要完整
- **每次 Build**：每加一个页面就 Build 一次，积压错误难排查
